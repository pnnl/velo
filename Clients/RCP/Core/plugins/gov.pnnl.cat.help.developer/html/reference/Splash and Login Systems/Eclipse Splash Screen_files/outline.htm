Currently in Eclipse
  • The Eclipse application defines the splash screen in it’s config.ini file.


  • The splash file must be of type .bmp and have the name “splash” (i.e. splash.bmp).  This is not configurable since it is defined in the Product Branding plug-in extension itself.


  • The config.ini file allows the user two different ways of specifying the splash files location.
  • Either as a common-separated list of locations to search;
  •                  osgi.splashPath = platform:/base/plugins/org.eclipse.platform
  • Or, as a direct location;
  •          osgi.splashLocation = platform:/


  • Note: The current location for the splash file is within the folder “eclipse\plugins\org.eclipse.platform_3.1.2” therefore I am assuming from the above search path that anything past the underscore (the version number) is ignored.


  • Note: If both configurations are defined the osgi.splashLocation takes precedent.


  • Note: When the search is used the finally resolved location is mapped to the osgi.splashLocation Key.

RCP and the Splash Screen
  • The Splash Screen is part of the plugin’s product branding and is not extended in a RCP Project by default.
  • A RCP application wanting a splash screen must be a product and therefore extend these extension points org.eclipse.core.runtime.products AND org.eclipse.core.runtime.applications.

How to do this.
  • We’ll need to create a class that builds a screen with the username and password prompts.
  • I added an authentication method to the Application class ( in our RCP application this is the IDEApplication.class file) to deal with the creation of the dialog box and to handle an invalid authentication.
  • I borrowed Uday’s LoginDialog.class (see references) and modified the code for our image and our RCP application.  Authentication is currently handled in this class but can be easily moved or changed to an external service.
